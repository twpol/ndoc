The MSDN and VS.NET documenters that ship with NDoc use a number of system attributes to augment the code comment tags when generating compiled help files. When a type or member is decorated with the attributes described below, NDoc will use this information in order to make the help file more descriptive.

Attribute Description

ObsoleteAttribute

Marks program elements that are no longer in use. NDoc will include warning text for each item that is marked as obsolete.

EditorBrowsableAttribute

This attribute prevents types of members from showing up in editor or type browsers. If the project setting EditorBrowsableFilter is set to true, then no documentation will be generated for items decorated with this attribute.

AssemblyVersionAttribute

If the IncludeAssemblyVersion project setting is true, then the assembly version will be included on each topic page.
